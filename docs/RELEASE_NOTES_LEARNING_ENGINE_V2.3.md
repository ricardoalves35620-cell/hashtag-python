# Learning Engine V2.3 — Logic, loops and data structures

## Goal

Replace generic learning prompts in phases 5–12 with concept-specific reasoning journeys. The learner first understands the problem and the data model, then writes pseudocode, maps it to Python, traces execution, debugs and transfers the idea.

## Dedicated migrations

- Phase 5: complete two-way decisions and boundary testing
- Phase 6: ordered multi-branch classification
- Phase 7: while-loop invariants, state updates and stopping conditions
- Phase 8: iteration over known collections
- Phase 9: nested data shape and two-level traversal
- Phase 10: dictionaries as named records and key/value contracts
- Phase 11: lists of dictionaries as table-like record collections
- Phase 12: list comprehensions derived from readable loops

## Pedagogical contract

Each migrated phase explicitly teaches:

1. the real problem;
2. the human solution;
3. inputs, state, rules and edge cases;
4. data flow;
5. pseudocode;
6. deliberate Python translation;
7. execution tracing;
8. likely failure and smallest confirming test;
9. practice planning;
10. transfer to a new problem.

Optional journal entries remain optional and never block progression.
