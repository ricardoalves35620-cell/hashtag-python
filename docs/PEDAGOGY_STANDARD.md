# Hashtag Python Pedagogy Standard

A phase is mastery-ready only when the learner must understand, predict, implement, debug and explain—not merely click Run.

## Required sequence

1. **Mental model** — explain what the concept represents.
2. **Worked example** — trace a minimal example line by line.
3. **Professional example** — show realistic use and trade-offs.
4. **Common failure** — demonstrate a plausible mistake and traceback.
5. **Guided reasoning** — Predict → Run → Change → Run again → Explain.
6. **Independent implementation** — visible and hidden behavioral tests.
7. **Hardening** — validation, naming, decomposition and error handling.
8. **Retrieval practice** — quiz questions covering concept, code reading and diagnosis.
9. **Generalization exam** — materially different hidden scenario.
10. **Reflection** — learner explains why the solution works and where it fails.

## Non-negotiable rules

- Running unchanged starter code cannot prove completion.
- Suggested inputs and expected output must come from the same source of truth.
- Hidden tests must change values and edge conditions, not only formatting.
- Feedback must explain the cause and next correction.
- Python syntax remains in English; instructional text supports Portuguese and English.
- A phase passing the automated audit still requires human review and learner observation.
