# Learning Engine V2.7 — Professional Python Journey

## Purpose

The Professional Python track, phases 28–39, now uses hand-authored reasoning blueprints. The learner is taught how professional tools protect change, reproducibility, contracts and evidence—not merely which commands or syntax to copy.

## Individually migrated phases

- 28: project anatomy, responsibilities and the minimum useful structure;
- 29: virtual environments, interpreter identity and reproducible dependencies;
- 30: modules, explicit imports, dependency direction and import side effects;
- 31: packages, `src` layout, deliberate public APIs and installed imports;
- 32: command-line interfaces as replaceable application boundaries;
- 33: Git as intentional, reviewable and reversible change history;
- 34: pytest, behavioral contracts, boundaries, failure cases and regression evidence;
- 35: tracebacks, reproducible failures and hypothesis-driven debugging;
- 36: safe logging, configuration validation, correlation and secret redaction;
- 37: type hints, dataclasses, runtime validation and meaningful contracts;
- 38: object invariants, composition, collaborator injection and restrained inheritance;
- 39: a professional capstone combining packages, types, tests, CLI, persistence, logging and reproducible delivery.

## Pedagogical result

Every professional phase now leads the learner through:

1. a concrete maintenance or delivery problem;
2. the human mental model behind the tool;
3. responsibility and boundary decomposition;
4. the real data or change flow;
5. an ordered professional workflow before syntax;
6. translation to the Python and tooling already authored in the phase;
7. trace-based reasoning;
8. realistic failure diagnosis;
9. deliberate practice planning;
10. transfer to a different project.

## Progression and access

No owner bypass or developer unlock is included. The normal course progression remains the learner experience, while automated agents provide complete locked-curriculum audit coverage.

## Validation target

- phases 0–39 report dedicated Learning Engine V2 blueprints;
- phase 40 remains pending the next migration batch;
- generic journey copy is absent from phases 28–39;
- the Professional Python capstone explicitly covers reproducibility, replaceable boundaries and executable evidence.
