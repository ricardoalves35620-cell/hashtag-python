# Sprint 2 — Learning quality and career readiness

This sprint moves the product from phase completion toward demonstrable mastery.

## Delivered

- A learning contract above every exercise: objective, time, difficulty, skills, evidence of success, common mistakes and workplace relevance.
- Optional authoring metadata for exercises without breaking legacy content.
- Sensible inferred metadata for all existing exercises.
- A Career Readiness page that maps passed exams to portfolio evidence and job-oriented milestones.
- Portfolio project expectations for foundations, professional Python, engineering and local AI.
- Automated tests for the pedagogical metadata layer.

## Product rule

A learner is not considered ready because a screen was completed. Readiness requires behavior that can be tested, explained and reproduced in a project.
