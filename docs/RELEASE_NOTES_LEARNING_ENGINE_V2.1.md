# Learning Engine V2.1

## Goal

Replace phase-clearing behavior with a reasoning-first learning contract. A learner should be able to explain, trace, debug, and transfer a concept before practice is treated as meaningful.

## What changed

Every one of the 69 phases now runs through ten distinct learning steps:

1. The challenge
2. Human intuition
3. Decomposition
4. Data flow
5. Pseudocode
6. Python translation
7. Line-by-line tracing
8. Debugging
9. Deliberate-practice planning
10. Explanation and transfer

Phases 0–4 received dedicated bilingual blueprints written for their individual concepts. Later phases use the same engine with their authored material and will receive dedicated blueprints in subsequent migrations.

## Product rule

A phase is not complete merely because code ran or a button was clicked. The learning flow requires a written reasoning checkpoint at every step and preserves the original authored code inside the Python-translation step.
