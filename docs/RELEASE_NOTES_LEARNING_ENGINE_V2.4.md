# Learning Engine V2.4 — From exercise to professional project

## Goal

Teach the complete development cycle instead of treating a large exercise as a project. A learner must understand the contract, plan the solution, implement it, prove behavior with multiple tests, and improve the code without breaking it.

## New mini-project workflow

Every project uses five checkpoints. At milestone phases, the project is part of phase mastery and the next phase remains locked until it is complete:

1. **Understand** — define inputs, output, business rules, and one edge case.
2. **Plan** — write ordered pseudocode before Python.
3. **Implement** — change the starter code and achieve an error-free run using the displayed suggested inputs.
4. **Test** — run the same program against multiple cases and required Python structures.
5. **Refactor** — select and apply at least one quality improvement, then rerun all tests.

Personal journal entries remain optional. The planning fields inside a project are required because they are project artifacts, not personal reflection.

## Foundation block projects

- Phase 4: **Damage Estimate** — input, conversion, variables, arithmetic, and formatted output.
- Phase 7: **Claim Queue Processor** — while-loop state, counter, accumulator, stopping condition, and empty input.
- Phase 12: **Claims Portfolio Report** — list of dictionaries, filtering, list comprehension, and aggregation.

Every displayed suggested input is verified against the evidence expected by the project tests.

## Persistence

Mini-project work is saved immediately in local storage. Authenticated learners can sync the same project state across devices after running:

```text
supabase/learning-project-progress.sql
```

Profile → Reset progress also removes mini-project state.
