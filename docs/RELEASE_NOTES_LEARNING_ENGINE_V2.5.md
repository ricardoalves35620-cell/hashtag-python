# Learning Engine V2.5 — Educational Workspace

## Purpose

The interface now reflects the learning method introduced by Learning Engine V2. Lessons are no longer displayed as generic content pages. They use a focused educational workspace with a journey map, readable content column, explicit current objective, optional journal, and persistent navigation.

## Main changes

- Responsive journey rail for the ten reasoning steps.
- Desktop two-column workspace with a sticky journey map and focused reading width.
- Mobile horizontal journey strip that does not compress the lesson content.
- New lesson hero with phase, current step, purpose, visited units and progress.
- Semantic learning callouts for key ideas, observations, common mistakes, professional habits and reminders.
- Optional Programmer Journal is clearly marked as optional and never blocks navigation.
- Sticky previous/next actions remain reachable without covering content.
- Every lesson change resets the page to the top and focuses the new lesson title.
- Phase overview is now a learning path with a clear next action, locked steps and mastery evidence.
- Mini-projects reuse the same professional checkpoint navigation and focused workspace.
- Raw dark-only colors were removed from the updated learning surfaces so light and dark themes use semantic tokens.

## Pedagogical principle

Progress represents evidence of learning, not the number of clicks. The interface must help the learner understand where they are, why the current activity exists, and what evidence is required next.
